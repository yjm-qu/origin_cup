# 方案v6A 实验记录

## 一、方案介绍

基于设计文档v6.0的方案A：卷积+池化+分组池化预处理，结合老师建议修改。

**任务**：二分类问题（MNIST手写数字4和9）

## 二、实现细节

| 批次大小 | 50 |
| 学习率 | 0.001 |
| 优化器 | Adam |

### 2. 数据预处理（方案A）

```
输入: 16×16 图像
    ↓
卷积层: Conv2d(1, 4, kernel=3) + ReLU + MaxPool2d
    ↓
输出: 4×8×8
    ↓
分组池化: 每个通道的8×8分成8个条，分别池化
    ↓
输出: 8维特征向量
```

### 3. 量子电路结构

每层包含：
1. **RY(encoding)** - 角度编码层：`angle = (π/16) × value`
2. **CRZ(theta)** - 可训练纠缠层（7个相邻CRZ门）
3. **RY(theta)** - 可训练层（8个RY门）

重复8层，最后测量第0个量子比特。

### 4. 损失函数

```python
MSE Loss: L = (y - P)²
```

其中 y 为标签(0或1)，P 为量子线路输出的概率。

### 5. 梯度计算

使用参数移位法则（Parameter Shift Rule）：
- 梯度 = coefficient × (F(θ+π/2) - F(θ-π/2)) / 2
- coefficient = -2 × (F - y)

## 四、存在的问题

### 问题1：损失值越训练越极端

**现象**：
- 训练初期损失有 0.6747 和 0.0319
- 训练后期损失变为 0.0000 或 1.0000
- 无法看到平滑的收敛曲线

**原因分析**：
- MSE Loss 在二分类问题上，当标签为0/1时，损失只能是 0 或 1 的平方
- 当模型输出 P 接近 0 或 1 时，梯度接近 0，导致参数在两个极端之间跳动

**可能的解决方案**：
1. 改用 BCE Loss（二分类交叉熵）
2. 调整学习率
3. 检查梯度计算是否正确

### 问题2：uniqc库噪声模型问题

**现象**：无法使用双比特门去极化噪声

**原因**：
- uniqc库的 `TwoQubitDepolarizing` 在密度矩阵模式下有bug
- 报错：`TypeError: 'int' object is not subscriptable`

**影响范围**：
| 门 | 状态 |
|----|------|
| CNOT | ❌ 报错 |
| CZ | ❌ 报错 |
| CP | ✅ 可用（但未正确加噪声） |
| XX (RXX) | ❌ 报错 |
| YY (RYY) | ❌ 报错 |
| ZZ (RZZ) | ❌ 报错 |

**读取噪声**：正常工作 ✅
